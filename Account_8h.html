Detailed Description

Declaration of the Account and Role entities for the Zoo Management System.

Author
Denys Freyuk
Date
27.10.2025
Version
1.1

This header defines the Account class and Role enumeration used to represent authentication and authorization data in the Zoo Management System. Each account has a username, hashed password, and a specific role (ADMIN, MANAGER, EMPLOYEE). The class provides utility methods for serialization, deserialization, and role conversions.

enum class Role
strong

Represents user access levels within the Zoo Management System.

The Role enumeration defines the hierarchy of user privileges: administrators have full access, managers handle staff and operations, and employees have limited access for daily tasks.

Enumerator
ADMIN 

Full system access and administrative privileges.

MANAGER 

Managerial access — can modify employee and animal data.

EMPLOYEE 

Standard employee access — limited operational rights.
